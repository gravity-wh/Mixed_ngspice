🔬 关键对比: 本方案 vs. "全部 FP32"

如果把整个 BSIM4v5 全部改为 FP32 (不保留任何 FP64 孤岛), 会发生什么:

计算步骤全部 FP32 的结果本方案 (混合精度)
Leff = L − 2·dl45nm − 1.2nm → FP32 消去误差 → Leff 偏差 >5%(double) 计算 → 精确
Vbi = Vt·ln(nsd·ndep/ni²)nsd·ndep overflow → inf → Vbi=inf → Vth=NaN(double) 计算 → 精确
ΔVth = k1ox·√φs − k1·√φ两项近乎相等相减 → FP32 精度丢失 → Vth 偏差(double) 计算 → 精确
噪声 NOIA 默认值6.25e41 overflow → inf → 1/f 噪声 NaN(double) 计算 → 精确
Vbseff sqrtsqrt 消去 → 体效应偏移(double) 计算 → 精确

全部 FP32 = 立即 NaN。实验证明: 热路径局部变量改为 float → 收敛前 NaN。这 5 个 FP64 精度孤岛是方案能工作的必要条件, 总共 ~50 行 double 运算/晶体管, < 0.1% 额外开销。
