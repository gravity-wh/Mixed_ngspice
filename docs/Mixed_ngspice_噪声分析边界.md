# Mixed_ngspice FP32 — 噪声分析边界研究

> 日期: 2026-07-17 | 首个 FP32 明确失败的分析维度

---

## 一、发现

5T OTA 噪声分析 (PTM 45nm LP, TT corner, `fnoimod=1`)：

| 频率 | FP32 | FP64 | FP32/FP64 | 现象 |
|------|------|------|-----------|------|
| 1Hz | 3.06e-6 | 1.04e-3 | **0.3%** | 1/f 噪声完全丢失 |
| 10Hz | 3.06e-6 | 3.29e-4 | **0.9%** | 同上 |
| 1kHz | 3.06e-6 | 3.32e-5 | **9.2%** | 同上 |
| 100kHz | 3.06e-6 | 5.25e-6 | 58% | 过渡区 |
| 1MHz | 3.05e-6 | 4.19e-6 | 73% | 接近热噪声 |
| 10MHz | 2.35e-6 | 2.73e-6 | 86% | 热噪声近似正确 |

**积分噪声 (1-10MHz)**：FP32 = 8.76e-3 Vrms，FP64 = 1.18e-2 Vrms → FP32 低估 25.6%

## 二、根因分析

BSIM4 的 flicker (1/f) 噪声模型 (`fnoimod=1`) 需要内部计算噪声参数。PTM 45nm 模型卡中未显式给出 `noia/noib/noic/kf` 等参数，BSIM4 从其他物理参数（tox, Cox, gm 等）推导默认值。

推导线中涉及：
```
S_id_flicker = KF · Ids^AF / (Cox · Leff^2 · f^EF)
```

其中 `Cox = εox/tox`，tox ≈ 1.8e-9。Cox ≈ 3.9×8.85e-12/1.8e-9 ≈ 0.019 F/m²。在 FP32 下，与极小尺寸（Leff=45nm, Weff=1µm）相乘后，中间量可能下溢到 FP32 精度地板以下 → 1/f 噪声电流归零。

**FP32 下只有热噪声保留**——因为热噪声公式 `S_id_thermal = 4kT · (gm + gds + ...)` 不涉及极小量的乘除。

## 三、影响评估

| 电路类型 | 噪声需求 | 影响 |
|----------|----------|------|
| OTA/运放 (低频) | 1/f 噪声主导 offset 和 SNR | ❌ FP32 低估 100-300× |
| 比较器 (瞬态噪声) | 1/f + 热噪声混合 | ⚠️ 低频分量缺失 |
| LDO (噪声 PSR) | 宽带噪声 | ⚠️ 总噪声低估 ~25% |
| RF 电路 (>1MHz) | 热噪声主导 | ✅ 近似可用 |

## 四、候选修复方案

### 方案 A：噪声参数显式化（最小改动）

在 PTM 模型卡中显式添加 FP32-safe 的噪声参数：

```spice
+noia = 1e-3 noib = 5e4 noic = -1.4e-12
+noimod = 1
```

绕过 BSIM4 内部推导，直接使用显式值，避免 FP32 下溢。

### 方案 B：噪声计算 FP64 孤岛（参考 Vbi 修复）

在 BSIM4 噪声计算函数中加入 `(double)` 强制提升，类似 patch 012 (vbi overflow) 的做法。改动量 ~5 行。

### 方案 C：分频段混合精度

低频 (<1kHz) 用 FP64 噪声计算，高频用 FP32。接口复杂度较高。

## 五、当前建议

噪声分析标记为 **FP64-only 特性**，不在 FP32 混合精度的宣称范围内。需要噪声分析时，使用 FP64 参考二进制。这是 FP32 混合精度 ngspice 的已知限制，不影响 DC/AC/TRAN 的正确性。
