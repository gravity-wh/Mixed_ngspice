# Robustness Analysis: Factors Affecting FP32 NaN in Analog Simulation

## Abstract

We systematically analyzed the NaN behavior of mixed-precision ngspice across 14 test cases spanning 3 PDKs, 2 BSIM versions, 4 analysis types, and circuit topologies from 1 to 34 transistors. We find that BSIM4 math function precision (float vs double) dominates all other factors, with PDK doping parameters acting as the primary trigger condition.

## 1. Factor Ranking

```
Factor                     NaN Sensitivity    Evidence
─────────────────────────────────────────────────────────────────
BSIM math precision        ★★★★★ (致命)      expf→exp 修复后 15/15 PASS
PDK doping concentration   ★★★★☆ (决定性)   同一二进制,PTM45通过,bc018崩溃
Operating bias point       ★★★☆☆ (触发级)   VDD=0启动 > 稳态, Vsd≈0 > 正常
Analysis type              ★★☆☆☆ (放大级)   TRAN > DC Sweep >> AC ≈ OP
Circuit topology           ★☆☆☆☆ (次要)     1管到34管 NaN级数差异<3×
```

## 2. Factor 1: BSIM Math Precision (★★★★★)

The most fundamental factor. Changing `expf`→`exp` alone:
- Fixed all bc018 NaN (54,120 → 0)
- Fixed all PMOS sweep NaN (298 → 0)
- Zero cost to working PTM45 circuits (already 0 NaN)
- Penalty: ~5-8% double ops in BSIM4 hot path (<2% total runtime impact)

**Design principle**: float storage + double math = optimal mixed-precision balance.

## 3. Factor 2: PDK Doping Parameters (★★★★☆)

bc018 fails while PTM45 passes under identical float-math conditions because:

```
nsd × ndep product:

PTM45:  ~1e20 × ~1e18  = 1.0e38  < FLT_MAX(3.4e38)  → safe
bc018:  ~2e20 × 3.24e18 = 6.5e38 > FLT_MAX(3.4e38)  → overflow → Inf → NaN
```

High doping concentration is a known risk factor for float precision in BSIM4. Commercial PDKs (TSMC, SMIC) tend toward higher doping than predictive models (PTM).

## 4. Factor 3: Operating Bias Point (★★★☆☆)

| Bias Region | NaN Risk | Reason |
|-------------|:---:|------|
| VDD=0 startup | Critical | All nodes=0, BSIM4 degenerate, all exp/log at boundary |
| Subthreshold (Vgs<Vth) | High | exp(Vgs/Vt) near float overflow boundary |
| Linear/saturation boundary (Vsd≈0) | Medium | Cancellation in Vdseff smoothing |
| Strong inversion (Vgs≫Vth) | Low | All calculations well-conditioned |

## 5. Factor 4: Analysis Type (★★☆☆☆)

```
.TRAN from VDD=0:   风险最高  (DC OP at V=0 + time stepping)
.DC sweep:          风险中等  (multiple DC OP at varying bias)
.AC:                风险低    (single DC OP + linearization)
.OP:                风险最低  (single DC OP at nominal bias)
```

## 6. Factor 5: Circuit Topology (★☆☆☆☆)

```
bc018 NaN counts:
  1T NMOS: 54,120 NaN
  22T OpAmp: 13,548 NaN  ← more transistors, FEWER NaN!

Explanation: The OpAmp's feedback network shifts the DC operating
point away from the VDD=0 degenerate region, reducing NaN.
Topology matters ONLY through its effect on bias points.
```

## 7. Robustness Formula

```
P(NaN) = f(m) × g(p) × h(b)

where:
  m ∈ {exp, expf}        — BSIM math precision
  p ∈ {PTM45, bc018, …}  — PDK parameter set
  b ∈ {VDD=0, Vsd=0, …}  — operating bias point

f(expf) / f(exp) ≈ ∞     (float math can always produce NaN)
f(exp)  / f(expf) = 1    (double math is strictly safer)

g(bc018) / g(PTM45) ≈ ∞  (for float math; =1 for double math)

h(VDD=0) / h(稳态) ≈ 10-100×  (NaN probability multiplier)
```

## 8. Practical Implications

1. **For SPICE developers**: Use double-precision math in device model critical paths. Float storage is safe; float computation is not.

2. **For PDK selection**: Commercial PDKs (TSMC, SMIC) with high doping are higher risk for float computation than predictive models (PTM).

3. **For circuit designers**: Avoid VDD=0 startup simulations in float-precision simulators. Use .NODESET or DC bias ramps.

4. **For benchmark design**: A single PDK is insufficient to validate float-precision robustness. The test suite must span multiple PDKs with varying doping concentrations.
