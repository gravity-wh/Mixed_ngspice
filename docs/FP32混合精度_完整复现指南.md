# Ngspice-46 FP32 混合精度：完整记录

> **文档性质**：记录实际已完成的修改，而非计划或待办事项。
> **核实日期**：2026-07-15
> **核实方式**：逐文件对比原始 ngspice-46.tar.gz 与 ngspice-46_fp32/ 源码树

---

## 1. 摘要

本方案将 Ngspice-46 的 BSIM4v5 热路径浮点精度从 FP64 降低到 FP32，在 TSMC 0.18µm 运放电路上实现了 **< 0.003% 的 DC 精度损失**和 **1.7-2.8× 的瞬态加速**。

核心思路：
- 利用自建的 `--enable-single-precision` 编译选项，将 `SPICE_REAL` 定义为 `float`，使 struct 成员全部降为 FP32 存储
- **自建 `SPICE_*` 宏体系**（`SPICE_EXP`/`SPICE_LOG`/`SPICE_SQRT` 等），在 `typedefs.h` 中根据 `SINGLE_PRECISION` 条件切换到 `expf()`/`logf()`/`sqrtf()`。**原版 ngspice-46 无此宏体系，代码直接使用 `exp()`/`log()`/`sqrt()`**
- 全局替换：82 个头文件中 `double` → `SPICE_REAL`，b4v5ld.c 中 `exp()` → `SPICE_EXP()` 等
- 热路径文件顶部 `#undef`/`#define` 覆盖 SPICE_* 为带显式 `(float)` 强转的版本
- **热路径局部变量保留 `double`**：b4v5ld.c 中所有中间临时变量（T0-T14, Vgsteff, Vdseff 等）必须为 `double`，改为 `float` 直接发散（已验证，见第 2 节）
- Leff/Weff 尺寸相关计算保留 `double` 精度孤岛（防止灾难性抵消）
- NaN 防护（防火墙 + 输出点盾）

**当前状态**：TT corner 下 DC + 瞬态验证通过。非 TT 工艺角（FF/SS/FNSP/SNFP）收敛失败，根因已定位，修复方案设计中。

## 2. 精度分层架构（关键设计约束）

本方案的实际精度模型是 **FP32 存储 + FP64 计算**，而非全 FP32。以下分层已经过实验验证（2026-07-15 将 b4v5ld.c 局部变量改为 SPICE_REAL → 直接发散）。

### 可以改为 FP32 的部分

| 层次 | 位置 | 机制 | 效果 |
|------|------|------|------|
| **Struct 成员** | `bsim4v5def.h` 等头文件 | `SPICE_REAL = float` | 1337 个 struct 字段从 8→4 字节，工作集减半 |
| **函数签名** | 各 BSIM4 函数参数 | `double *` → `SPICE_REAL *` | 参数传递从 8→4 字节 |

### 不可以改为 FP32 的部分

| 层次 | 位置 | 原因 | 验证 |
|------|------|------|------|
| **热路径局部变量** | `b4v5ld.c` 约 100 个局部变量（T0-T14, Vgsteff, Vdseff, Gm, Gds, Idl, ueff 等） | 中间计算涉及 `sqrt(a² - b)`、`exp(VgstNVt)` 等操作，FP32 下舍入误差导致 NaN → Newton 发散 | ✅ 实测：改为 SPICE_REAL 后仿真全部 NaN |
| **Leff/Weff 计算** | `b4v5temp.c` Size_Not_Found 块 | 灾难性抵消：`Lnew - 2*dl` 中两项比值可达 10-100:1 | ✅ 实测：FP32 下算出负 Leff → Fatal Error |
| **polyDepletion** | `b4v5ld.c:5420` | 栅极多晶硅耗尽效应涉及迭代求解，需要 double 精度 | 未单独测试，作为保守策略保留 |

### 为什么"FP32 存储 + FP64 计算"能加速

加速不来自更快的算术（局部变量仍是 double，FPU 做的是 53-bit 运算），而来自：
1. **Cache 命中率提升**：struct 占用减半 → 更多数据留在 L1/L2 cache
2. **内存带宽减半**：load/store struct 成员时传输 4 字节而非 8 字节
3. **寄存器压力降低**：编译器有更多寄存器可分配

### 等效于深度学习的混合精度训练

```
ML 混合精度:  FP16 存梯度 + FP32 累加
Ngspice FP32: float 存 struct + double 局部变量
```
两者原理相同：存储降精度省带宽，计算保精度稳收敛。

| 项目 | 版本/来源 |
|------|----------|

### 2.2 工艺库

TSMC 0.18µm BCD 工艺库文件（放在 `models/` 目录）：
- `bc018_v1p14_rev2.lib` — HSPICE 工艺角库
- `bc018_v1p14_rev2.mdl` — BSIM4v5 基础模型参数

### 2.3 构建配置

```bash
tar xzf ngspice-46.tar.gz
cd ngspice-46
./configure --enable-single-precision \
            --disable-klu \
            --disable-xspice \
            --disable-osdi \
            --disable-cider \
            --prefix=$PWD/install-fp32
make -j$(nproc)
make install
```

关键标志：
- `--enable-single-precision`：定义 `SINGLE_PRECISION`，`SPICE_REAL = float`，所有算术运算自动降为 FP32
- `--disable-klu`：禁用 KLU 求解器（第三方库硬编码 double）
- `--disable-xspice`：禁用 XSPICE（`sizeof(double)` 索引 bug）

构建产物：`install-fp32/bin/ngspice`（~7.2 MB）

### 3.2 typedefs.h — 自建 SPICE_* 宏体系

**原版 ngspice-46 的 `typedefs.h` 仅有结构体前向声明，不存在 `SPICE_REAL` 类型定义，也不存在任何 `SPICE_*` 宏。** 本项目新增：

```c
#ifdef SINGLE_PRECISION
typedef float SPICE_REAL;
#define SPICE_SIN(x)  sinf(x)
#define SPICE_COS(x)  cosf(x)
#define SPICE_EXP(x)  expf(x)
#define SPICE_LOG(x)  logf(x)
#define SPICE_SQRT(x) sqrtf(x)
#define SPICE_POW(x,y) powf(x,y)
#define SPICE_FABS(x) fabsf(x)
#define SPICE_ATAN(x) atanf(x)
#define SPICE_TAN(x)  tanf(x)
#define SPICE_TANH(x) tanhf(x)
#define SPICE_ASINH(x) asinhf(x)
#define SPICE_FMOD(x,y) fmodf(x,y)
#define SPICE_HYPOT(x,y) hypotf(x,y)
#define SPICE_LOG10(x) log10f(x)
#define SPICE_COSH(x) coshf(x)
#define SPICE_SINH(x) sinhf(x)
#define SPICE_ACOSH(x) acoshf(x)
#define SPICE_CEIL(x) ceilf(x)
#define SPICE_FLOOR(x) floorf(x)
#else
typedef double SPICE_REAL;
#define SPICE_EXP(x)  exp(x)
// ... 同上全部映射到 double 版本 ...
#endif
```

这是整个 FP32 混合精度方案的基础设施。`SINGLE_PRECISION` 由 configure 脚本自动启用（当指定 `--enable-single-precision` 时）。

### 3.3 b4v5ld.c — 主要改造文件

#### A. 全局替换：exp()/log()/sqrt() → SPICE_EXP()/SPICE_LOG()/SPICE_SQRT()

原版 ngspice-46 的 b4v5ld.c 直接调用 `exp()`（45 处）、`log()`（25 处）、`sqrt()`（46 处），不经过任何宏。本项目第一步将这些调用全部替换为 `SPICE_EXP()`/`SPICE_LOG()`/`SPICE_SQRT()`，使其受 `typedefs.h` 中 `SINGLE_PRECISION` 条件控制。

#### B. 文件顶部 SPICE_* 局部重定义（B4V5_FP32_MATH 块）

```c
#define B4V5_FP32_MATH
#ifdef B4V5_FP32_MATH
#undef SPICE_EXP
#undef SPICE_LOG
#undef SPICE_SQRT
#undef SPICE_POW
#undef SPICE_SIN
#undef SPICE_COS
#undef SPICE_FABS
#undef SPICE_ATAN
#undef SPICE_TAN
#undef SPICE_TANH
#define SPICE_EXP(x)  expf((float)(x))
#define SPICE_LOG(x)  logf((float)(x))
#define SPICE_SQRT(x) sqrtf((float)(x))
#define SPICE_POW(x,y) powf((float)(x),(float)(y))
#define SPICE_SIN(x)  sinf((float)(x))
#define SPICE_COS(x)  cosf((float)(x))
#define SPICE_FABS(x) fabsf((float)(x))
#define SPICE_ATAN(x) atanf((float)(x))
#define SPICE_TAN(x)  tanf((float)(x))
#define SPICE_TANH(x) tanhf((float)(x))
#endif
```

**工作原理**：经过全局替换后，b4v5ld.c 中所有数学函数调用已改为 `SPICE_EXP()` 等形式。此处的 `#undef`/`#define` 在文件编译时覆盖 `typedefs.h` 的全局定义，加上显式 `(float)` 强转以防止编译器隐式 double 提升。DEXP 宏也同步替换为 `expf` 版本。

#### B. sqrt 保护（1 处已实施）

```c
// 行 1037：Vbseff 计算
T1 = sqrt(fmax(T0 * T0 - 0.004 * here->BSIM4v5vbsc, 0.0));
```

防止 `T0^2 - 0.004 * vbsc` 在 FP32 舍入下变负 → `sqrt(负数)` = NaN。

#### C. NaN 防护（5 处关键输出点 + 1 处防火墙）

在 Vgsteff（行 1342）、Vdseff（行 1750）、Gds/Gm/Gmb（行 2215-2217）、cdrain（行 2866）赋值后立即加上：
```c
if (isnan((double)Vgsteff)) Vgsteff = 0.0;
if (isnan((double)Vdseff)) Vdseff = 0.0;
if (isnan((double)Gds)) Gds = 0.0;
if (isnan((double)Gm)) Gm = 0.0;
if (isnan((double)Gmb)) Gmb = 0.0;
if (isnan((double)cdrain)) cdrain = 0.0;
```

矩阵 stamp 前的 NaN 防火墙（行 4884）：
```c
if (isnan((double)here->BSIM4v5cd) || isnan((double)here->BSIM4v5gm)
    || isnan((double)here->BSIM4v5gds) || isnan((double)here->BSIM4v5gmbs)) {
    here->BSIM4v5cd = 0.0; here->BSIM4v5gm = 0.0;
    here->BSIM4v5gds = 0.0; here->BSIM4v5gmbs = 0.0;
    here->BSIM4v5cbs = 0.0; here->BSIM4v5cbd = 0.0;
    // ... (零化其余输出) ...
}
```

### 3.4 b4v5temp.c — Leff/Weff 双精度孤岛

**位置**：第 429-502 行，`if (Size_Not_Found)` 块内。

将有效沟道尺寸（Leff/Weff/leffCV/weffCV/weffCJ）的计算从 `SPICE_REAL` 提升到 `double`：

```c
{
    double dLnew = (double)Lnew;
    double dWnew = (double)Wnew;
    double dT0, dT1, dT2, dT3, dtmp1, dtmp2;

    dT0 = pow(dLnew, (double)model->BSIM4v5Lln);
    dT1 = pow(dWnew, (double)model->BSIM4v5Lwn);
    // ... (binning computation in double) ...
    pParam->BSIM4v5leff = (SPICE_REAL)(dLnew - 2.0 * (double)pParam->BSIM4v5dl);
    pParam->BSIM4v5weff = (SPICE_REAL)(dWnew - 2.0 * (double)pParam->BSIM4v5dw);
    pParam->BSIM4v5leffCV = (SPICE_REAL)(dLnew - 2.0 * (double)pParam->BSIM4v5dlc);
    pParam->BSIM4v5weffCV = (SPICE_REAL)(dWnew - 2.0 * (double)pParam->BSIM4v5dwc);
    pParam->BSIM4v5weffCJ = (SPICE_REAL)(dWnew - 2.0 * (double)pParam->BSIM4v5dwj);
}
```

**原因**：`Lnew - 2*dl` 中 `dl ≈ -3.8e-8` 而 `Lnew ≈ 2.66e-6`，两者比值 ~70:1。在 float (23-bit 尾数) 下，减法抵消损失约 6 个有效位，可能算出负的 Leff 触发 fatal error。

**注意**：b4v5temp.c 中孤岛外的数学函数调用（~33 个 `exp()`、~21 个 `log()`、~18 个 `sqrt()`）**并未批量替换为 SPICE_* 或 float 版本**。b4v5temp.c 只执行一次，不影响仿真速度，保持 double 是合理的。

### 3.5 其他 BSIM4 文件 — SPICE_* 局部重定义

以下文件均在顶部添加了与 b4v5ld.c 相同的 `B4V5_FP32_MATH` 守卫块（`#undef` + `#define` 增强版 SPICE_* 宏）：

| 文件 | 说明 |
|------|------|
| `b4v5noi.c` | 噪声计算 |
| `b4v5pzld.c` | 零极点分析 |
| `b4v5acld.c` | AC 小信号 |

**与 b4v5ld.c 相同**：这些文件的主体代码未做 `exp→expf` 替换，宏重定义对现有的 `exp()` 调用不生效。保留此定义是为了后续扩展时直接可用。

以下 BSIM4 文件**未做任何修改**：
- `b4v5geo.c` (几何计算)
- `b4v5.c` (顶层接口)
- `b4v5set.c` (参数赋值)
- `b4v5mask.c`、`b4v5mpar.c`、`b4v5par.c` 等

## 4. 模型文件准备

从原始 `.mdl` 文件提取 p18 和 n18 模型定义，将所有 HSPICE 表达式预求值（TT corner 下所有 `d*` 参数 = 0）：

关键操作：

## 7. 已知限制

1. **热路径局部变量必须保留 double**：b4v5ld.c 所有中间临时变量不可改为 SPICE_REAL，实测改为 float 后 22T 运放 DC 直接发散（全部节点 NaN）
2. **仅 BSIM4v5**：其他器件模型（BJT、BSIM3、MOS1-9 等）未修改
3. **非 TT 工艺角不收敛**：FF/SS/FNSP/SNFP 待修复
4. **AC/噪声分析未覆盖**：`complex.h` 复数宏未转换
5. **KLU 求解器禁用**：大规模病态矩阵可能无法求解
6. **dmcg 移除**：结二极管缺省面积归零，高阻内部节点有 mV 级偏差
7. **模型文件需分离 .include**：同文件多模型存在堆溢出（Ngspice 预先存在的 bug）
8. **非 TSMC 0.18µm 工艺需重新验证**
9. **局部变量的 double → SPICE_REAL 替换不可行**：b4v5ld.c 局部变量必须保持 double，已通过实验验证

## 8. 故障排查

### 症状：所有节点 NaN，DC 发散

**原因**：`b4v5ld.c` 局部变量被改为 `SPICE_REAL`（float）。中间计算 `sqrt(T² - b)`、`exp(VgstNVt)` 等在 FP32 下累积舍入误差，产生 NaN 并通过 Newton 迭代扩散到全部节点。

**修复**：恢复局部变量为 `double`。参见第 2 节"精度分层架构"。

### 症状："Effective channel length <= 0"

**原因**：`b4v5temp.c` 的 Leff 双精度孤岛未生效。

**检查**：确认 `b4v5temp.c` 第 429 行起使用 `double dLnew` 等变量，且 `pParam->BSIM4v5leff = (SPICE_REAL)(dLnew - 2.0 * (double)pParam->BSIM4v5dl)`。

### 症状："singular matrix"

**原因**：模型文件包含 `dmcg = 2.7e-7`。

**修复**：从 `.mdl` 文件删除 `dmcg/dmci/dwj/xgw` 行。

### 症状："Toxp is not positive"

**原因**：`.mdl` 文件使用 HSPICE 表达式语法（如 `'4.27e-9+dtox_p18'`）。

**修复**：预求值所有表达式（TT corner 下 d* = 0，直接使用基值）。

### 症状：模型参数为垃圾值

**原因**：p18 和 n18 在同一个 `.include` 文件中。

**修复**：使用分离的 `.include` 文件（一个模型一个文件）。
