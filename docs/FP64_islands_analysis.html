孤岛 1: Vbi — 掺杂浓度乘积溢出 FP32

文件: b4v5temp.c · Patch: 012 · 类型: overflow

// FP32 (原始): nsd·ndep = 2e20 × 3.24e18 = 6.48e38
// 6.48e38 > FLT_MAX(3.4e38) → overflow → +Inf
vbi = Vtm0 * log(nsd * ndep / (ni * ni));  // log(Inf) = Inf

// FP64 孤岛: 全部转换为 double 后再计算
vbi = ((double)Vtm0 * log((double)nsd * (double)ndep
                / ((double)ni * (double)ni)));
变量PTM45 典型值float 范围结果
nsd2.0×10²⁰✅ 3.4×10³⁸
ndep3.24×10¹⁸✅ 3.4×10³⁸
nsd × ndep6.48×10³⁸❌ 溢出!→ +Inf
log(+Inf)→ +Inf
Vbi = +Inf→ Vth = NaN

为什么这很隐蔽: nsd 和 ndep 各自都在 FP32 范围内（~10²⁰），单独存储没问题。只有乘在一起才溢出。BSIM4 公式直接用 nsd * ndep / (ni * ni) 作为 log() 的参数——没有任何中间变量存储这个乘积——但 FP32 的寄存器运算同样受 3.4×10³⁸ 限制。

为什么 (double) 转换就解决了: double 最大值是 1.8×10³⁰⁸，比 6.48×10³⁸ 大 270 个数量级。(double) 转换让乘法在 64 位寄存器中完成，最后 (SPICE_REAL) 截断存储结果——而 log(6.48e38) ≈ 89.4，远小于 3.4×10³⁸，截断回 float 完全安全。

孤岛 2: Vth k1ox — 近等数相减的消去误差

文件: b4v5ld.c · Patch: 013 · 类型: 消去误差

ΔVth = k1ox · √φs − k1 · √φ0 
// FP32:
Delt_k1ox = k1ox * sqrtPhis - k1 * sqrtPhi;
// ≈ 30 × 0.55 − 0.4 × 0.50 = 16.5 − 0.2 = 16.3

// FP64 孤岛:
double d_diff = (double)k1ox * (double)sqrtPhis - (double)k1 * (double)sqrtPhi;
Delt_k1ox = (SPICE_REAL)d_diff;
项典型值FP32 表示有效位
k1ox · √φs~16.51.65×10¹7 位
k1 · √φ0~0.22.0×10⁻¹7 位
差值~16.31.63×10¹仅 3 位 (丢失 4 位)

根因: k1ox ≈ 30–50, k1 ≈ 0.3–0.5, sqrtPhis ≈ sqrtPhi ≈ 0.5–0.6。第一项 ~15–30，第二项 ~0.15–0.3。两项相差 ~100 倍，相减时 FP32 只有 ~3 位有效数字（23 位尾数，但需要对齐指数后再减，丢失了 ~4 个十进制有效位）。Vth 偏移 1–2mV 就会影响整个 gm/gds/ids。

为什么用 double: double 有 52 位尾数（FP32 的 23 位），相差 100 倍的两个数相减仍有 ~13 个有效十进制位。

孤岛 3: Abulk — 最大孤岛 (35 行, 18 个 const double 输入)

文件: b4v5ld.c:1488 · Patch: 003 · 类型: 消去误差 · 规模: 最大孤岛

Abulk = 1 + T₁ · T₂
T₁ = ½·k1ox·LpeVb/√φs + k2ox − k3b·VthNW
T₂ = a₀ · (Leff / (Leff + 2√(xj·Xdep))) + b₀ / (weff + b₁) 
// FP64 孤岛: 18 个输入显式 (float)→const double 转换, 12 个 double 中间变量
const double d_k1ox = (float)pParam->BSIM4v5k1ox;
const double d_Leff = (float)Leff;
const double d_Xdep = (float)Xdep;   // ... 18 个输入
double d_T9, d_T1, d_T2, d_T5, d_T0, d_T8, d_T10;  // 12 个中间变量
double d_Abulk0, d_Abulk, d_dA0_dVb, d_dA_dVg, d_dA_dVb;

// 关键消去点 1: Leff / (Leff + 2√(xj·Xdep))
d_T9 = sqrt(d_xj * d_Xdep);
d_T5 = d_Leff / (d_Leff + 2.0 * d_T9);  // ← Leff 消去

// 关键消去点 2: a0·T5 + b0/(weff+b1)

为什么这是最大孤岛: Abulk 是 BSIM4 体效应因子的完整计算链。涉及 18 个模型参数 + 5 个运行期变量，运算链长（~35 行）。T₁ 中的 ½·k1ox/√φs 和 T₂ 中的 Leff/(Leff+2√(xj·Xdep)) 都有消去风险，且所有导数项 (dT1_dVb, dT2_dVb, dA0_dVb) 也需要 double 精度才能保证后续 Jacobian 矩阵的质量。

孤岛 4: Vbseff/Phis — sqrt 内的平方差消去

文件: b4v5ld.c · Patch: 003 · 类型: 消去误差

Vbseff = vbsc + ½(T₀ + √(T₀² − 0.004·vbsc))
其中 T₀ = Vbs − vbsc − 0.001 
// FP32: T₀² − 0.004·vbsc → 两个接近相等的数相减 → 消去误差
T0 = Vbs - vbsc - 0.001;      // Vbs ≈ vbsc → T0 很小
T1 = sqrt(T0*T0 - 0.004*vbsc); // T0² 和 0.004·vbsc 可能接近

// FP64 孤岛: 全部在 double 精度下计算
double dT0 = (double)Vbs - (double)vbsc - 0.001;
double dT1 = sqrt(fmax(dT0 * dT0 - 0.004 * (double)vbsc, 0.0));

当 Vbs ≈ vbsc 时（体效应电压接近衬底偏置），T₀ → 0。T₀² 和 0.004·vbsc 的值接近，FP32 下平方差消去会导致 sqrt 的参数变负或严重不准确。

为什么 oxideTrapDensity 值这么大: 这不是错误——是物理含义导致的。BSIM4 的 1/f 噪声模型用 Noia 表示陷阱密度（单位 cm⁻³eV⁻¹）。硅/SiO₂ 界面的陷阱密度本来就极高（~10¹⁰–10¹² cm⁻²），换算成体密度后达到 10⁴¹ 量级。这个值作为中间参数参与计算，最终噪声电流在合理的 pA–nA 范围。

开销: ~52 double 运算/频率点 × ~80 频点 × ~5 器件 = ~20,000 double 运算/噪声分析，< 一次 DC 分析计算量的 0.1%。

孤岛 6: Leff/Weff — 沟道长度消去

文件: b4v5temp.c · Patch: 004 · 类型: 消去误差

Leff = Lnew − 2×dl
其中 dl = Lint + Ll/LnewLln + Lw/WnewLwn + Lwl/(LnewLln×WnewLwn) 
// FP64 孤岛: 整个 binning 计算在 double 中进行
double dLnew = (double)Lnew;                    // Lnew ≈ 45nm = 4.5e-8
double dT0 = pow(dLnew, (double)model->Lln);     // pow 跨度 >20 量级
double dtmp1 = model->Ll / dT0 + model->Lw / dT1 + model->Lwl / (dT0*dT1);
pParam->dl = (SPICE_REAL)(model->Lint + dtmp1);  // dl ≈ 0.6nm
pParam->leff = (SPICE_REAL)(dLnew - 2.0 * pParam->dl);  // 45nm − 1.2nm
计算步骤值FP32 相对误差FP64 相对误差
Lnew4.5×10⁻⁸~10⁻⁷~10⁻¹⁶
dl6.0×10⁻¹⁰~10⁻⁷~10⁻¹⁶
Lnew − 2×dl4.38×10⁻⁸~5×10⁻³ (0.5%)~10⁻¹⁶
Leff 影响 gmgm ∝ 1/Leff → ~0.5%可忽略

为什么这么敏感: dl 的计算涉及 pow(Lnew, Lln)——Lln 是 binning 指数，通常 −0.5 到 −2.0 之间。45nm⁻² = 4.9×10¹⁴。Ll/Lnew^Lln 的除法跨越 20+ 个数量级。任何一个中间步骤的 FP32 舍入都会在最终的 Leff 中放大为 ~0.5% 的偏差——然后再通过 gm∝1/Leff 传播到所有小信号参数。
