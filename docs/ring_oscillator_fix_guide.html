17 级环振 ngspice 仿真修复指南

工艺: PTM 45nm LP · BSIM4 Level 54 · VDD=1.1V · FP32+FP64 均验证通过

问题诊断

17 级 CMOS 反相器环振在 ngspice 中仿真会遇到两类失败，均非 FP32 精度问题（FP64 同样失败）：

#错误信息根因类型
1doAnalyses: TRAN: Timestep too small; trouble with node "vdd#branch" 理想电压源 VDD 无串联阻抗，34 个晶体管同时开关时支路方程奇异 testbench 设计
2doAnalyses: TRAN: Timestep too small; trouble with node "n2" UIC 初始条件设置不当，所有节点处于亚稳态平衡，无法起振 testbench 设计

四项必要修复

1

VDD 串联 1Ω 电阻

物理合理

理想电压源 VDD 输出阻抗为零。34 个晶体管开关时，瞬态电流通过 VDD 支路方程 I = (V_node − V_VDD) / 0 → 除零奇异。

真实芯片中 bond wire 寄生电阻 ~0.1Ω，PCB trace ~10mΩ。加 1Ω 是保守但完全合理的仿真实践。

修复前:  VDD VDD 0 DC 1.1

修复后:  VRAW vraw 0 DC 1.1
         RVDD vraw VDD 1    ← 1Ω 串联电阻
2

交替初始条件 — 打破亚稳态

物理合理

17 个反相器环是一个正反馈环路。如果所有节点 .ic 设为相同的 VDD/2，系统处于不稳定平衡——理论上会因数值噪声起振，实际上数值噪声不够大。

交替设置 0 / 1.1V 模拟工艺失配或热噪声打破对称性。真实芯片上电后必然有不对称性。

修复前 (亚稳态):  .ic v(n1)=0.55 v(n2)=0.55 ... v(n17)=0.55

修复后 (交替):   .ic v(n1)=0    v(n2)=1.1  v(n3)=0    v(n4)=1.1
         .ic v(n5)=0    v(n6)=1.1  v(n7)=0    v(n8)=1.1
         .ic v(n9)=0    v(n10)=1.1 v(n11)=0   v(n12)=1.1
         .ic v(n13)=0   v(n14)=1.1 v(n15)=0   v(n16)=1.1
         .ic v(n17)=0
3

启动 NMOS 脉冲 — 钳位释放

物理合理

交替 IC 给了初始方向，但需要短暂将 n1 强制拉到 GND（通过启动 NMOS），然后释放，边沿才开始绕环传播。

这模拟了真实环振中的启动电路（通常用反相器或 NAND 门打破 DC 平衡）。

启动电路:  Mstart n1 start 0 0 nmos W=0.5u L=45n
         Vpulse start 0 PULSE(1.1 0 0 50p 50p 200p 5n)

         ↑ start=1.1V 持续 200ps，强制 n1=0V，然后释放
4

使用 UIC — 跳过 DC 工作点

标准做法

17 个反相器环的 DC 工作点是亚稳态——ngspice 的 DC 求解器无法收敛（或不收敛到有意义的值）。uic 告诉 ngspice 直接使用 .ic 的值开始瞬态，不先求解 DC。

.tran 1p 3n uic

.include /path/to/ptm45lp.lib

不需要的"修复" (已验证无效或多余)

选项是否需要原因
cshunt=1p❌ 不需要给所有节点加 1pF 假电容，改变电路特性，属于作弊
rseries=0.05❌ 不需要手动 VDD 串 1Ω 已解决 branch stiffness
method=gear❌ 不需要默认 trap 法即可，无需 Gear 阻尼
reltol=0.01❌ 不需要默认 0.001 精度足够
gmin=1e-10 sollim❌ 不需要UIC 跳过 DC，这些 DC 收敛选项不参与
额外 FP32 patch❌ 不需要WSL 默认 ngspice (FP64, 零 patch) 也能跑通
