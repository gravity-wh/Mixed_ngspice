方案 1: Vbi — 对数恒等式拆分 ✅ 已实施

文件: b4v5temp.c:1179 · 分类: overflow · 变更: 2 行 → 1 行

Vbi = Vtm0 · ln(nsd · ndep / ni²)

问题: nsd·ndep = 2×10²⁰ × 3.24×10¹⁸ = 6.48×10³⁸ > FLT_MAX(3.4×10³⁸) → overflow

Before (FP64 孤岛)
pParam->BSIM4v5vbi = (SPICE_REAL)(
  (double)Vtm0 * log(
    (double)nsd * (double)ndep
    / ((double)ni * (double)ni)
  )
);
After (纯 FP32)
pParam->BSIM4v5vbi = Vtm0 * (
  logf(nsd)
  + logf(ndep)
  - 2.0f * logf(ni)
);

原理: ln(a·b/c²) = ln(a) + ln(b) − 2·ln(c)。每个对数参数都远小于 FLT_MAX，不会溢出。这是数学恒等式，不引入任何近似——FP64 和 FP32 结果的差异仅来自浮点舍入（~2×10⁻¹⁶）。

方案 2: Vbseff — 平方差因式分解 ✅ 已实施

文件: b4v5ld.c:1061 · 分类: 消去 · 变更: ~25 行 → ~10 行 (两个块)

T1 = √(T₀² − 0.004·vbsc)   where   T₀ = Vbs − vbsc − 0.001

问题: 当 T₀ → 0（Vbs≈vbsc），T₀² 和 0.004·vbsc 接近相减 → 有效位数丢失。

Before (FP64 孤岛)
double dT0 = (double)Vbs
           - (double)vbsc - 0.001;
double dT1 = sqrt(fmax(
  dT0*dT0
  - 0.004*(double)vbsc, 0.0));
After (纯 FP32)
float sqrtC = sqrtf(0.004f * vbsc);
float fT0 = Vbs - vbsc - 0.001f;
float fT1 = sqrtf(fmaxf(
  (fT0 - sqrtC) * (fT0 + sqrtC),
  0.0f));

原理: T₀² − C = (T₀−√C)·(T₀+√C)。当 T₀≈√C 时，不再有近等数相减——√(小×大) 比 √(小−小) 数值稳定得多。数学恒等式，不引入近似。

同时替换: JX 正向体偏置修正块（相同方法）。

方案 3: Vth k1ox — Dekker 精确减法 ✅ 已实施

文件: b4v5ld.c:1207 · 分类: 消去 · 变更: 4 行 → 6 行

ΔVth = k1ox·√φs − k1·√φ0   (a/b ≈ 100×, 两数相差两个数量级)

问题: k1ox≈35, k1≈0.4, 两项差 ~100 倍。FP32 减法需要对齐指数，丢失 ~4 个有效十进制位。

Before (FP64 孤岛)
double d_diff =
  (double)k1ox * (double)sqrtPhis
  - (double)k1 * (double)sqrtPhi;
Delt_k1ox = (SPICE_REAL)d_diff;
After (纯 FP32 · 6 运算)
float a = k1ox * sqrtPhis;
float b = k1  * sqrtPhi;
float s = a - b;   // 近似差值
float t = a - s;   // 实际被减去的量
float c = t - b;   // 补偿项（丢失的位）
Delt_k1ox = s - c; // 补偿后的精确结果

原理: Dekker (1971) 证明了当 |a| ≥ |b| 时，这 6 个 FP32 运算能恢复普通减法丢失的所有尾数位。误差 ≤ 2u（u 为机器精度），等效于在 FP32 中得到 FP64 精度的减法结果。

方案 4: 噪声 NOIA — log-space 存储 ⬜ 待实施

未实施原因: 噪声孤岛仅在 .noise 分析时调用，不影响 DC/AC/TRAN 性能。当前噪声路径的 ~39 次 double 运算仅占单次噪声分析的 ~0.01%。转换优先级最低。

方案: struct 存储 log(NOIA) = 96.2 代替 NOIA = 6.25×10⁴¹（→+Inf），噪声计算用 exp(log_sum) 替代乘积链。
