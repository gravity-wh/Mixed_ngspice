# Pure FP32 Patch Set

Generated by `scripts/gen_pure_fp32_patches.py` from the main `patches/` directory.

## Purpose

These patches produce a **Pure FP32** (all-float, no double-precision islands) build of ngspice-46.
This serves as the "strawman" control group in the 3-way ablation study:

| Build | Patches | SPICE_REAL | Math | Islands |
|-------|---------|------------|------|---------|
| FP64  | All 15 (no-ops) | double | exp(), sqrt() | N/A |
| Mixed FP32 | All 15 | float | expf(), sqrtf() | double-precision islands active |
| **Pure FP32** | **10 patches** | **float** | **expf(), sqrtf()** | **none** |

## Excluded Patches (5)

These patches exclusively add double-precision safety nets. They are excluded
because Pure FP32 intentionally lets these overflow/underflow to demonstrate
the failure modes:

- `012-b4v5temp-vbi-overflow-fix.patch`
- `013-multi-t-nan-fix.patch`
- `015-noise-fp64-island.patch`

## Transformation Details

The `transform_patch()` function in `gen_pure_fp32_patches.py`:
1. Finds blocks delimited by `/* DOUBLE PRECISION ISLAND */` comments
2. Converts `double d_*` variable declarations to `float f_*`
3. Converts `(double)` casts to `(float)`
4. The FP32 math blocks (Vth, Vgsteff, Weff, Rds, Vdsat, Vdseff, Idl, VADIBL,
   VASCBE, Source Velocity Limit) already use `float f_*` and are preserved as-is.

## Expected Behavior

- DC: Works for basic circuits (single NMOS/PMOS), but with larger errors than Mixed FP32
- Vbi: May produce Inf for PTM 45nm (nsd*ndep/ni^2 > FLT_MAX)
- Noise: Flicker noise produces NaN (oxideTrapDensity > FLT_MAX)
- Non-TT corners: Earlier convergence failure than Mixed FP32
